2. Executive Summary of what happened:
On April 9, 2026, Falco detected suspicious activity in the Kubernetes cluster related to unauthorized cryptocurrency mining containers. 
During the investigation, three suspicious workloads were identified: moneropool, minergate, and cryptotonight. 
These workloads appeared to be running without authorization and represented a potential security incident.

3. Summary of what was affected:
The Kubernetes cluster was affected by the presence of three unauthorized pods. These workloads consumed cluster resources and introduced security concerns 
because they were not part of the expected application environment.

4. Summary of the impact:
The suspicious pods used CPU resources that should have been available for legitimate workloads. This could reduce cluster performance and increase infrastructure costs. 
Their presence also indicated a security issue in the environment because unauthorized workloads were able to run in the cluster.

5. Summary of the remediation steps:
The suspicious pods were identified and removed from the cluster to stop unauthorized resource usage. After removal, the cluster was reviewed to confirm 
that no additional suspicious workloads were running. Falco alerts were used as evidence during the investigation, and cluster security controls were reviewed 
for possible improvement.

6. Summary of lessons learned:
This incident showed the value of runtime security monitoring in Kubernetes environments. Falco helped detect suspicious activity quickly and made investigation easier. 
In the future, stronger preventive controls such as stricter RBAC policies, image validation, and admission control should be considered to reduce the risk of unauthorized 
workloads being deployed.
